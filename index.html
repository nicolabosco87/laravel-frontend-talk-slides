Introducing Laravel

Laravel è un framework open source di tipo MVC scritto in PHP per lo sviluppo di applicazioni web, creato nel 2011 da Taylor Otwell come derivazione di Symfony.

Attualmente alla versione 5.3 è uno dei framework PHP più utilizzati al momento.

Little Laravel overview

  • MVC framework with bundles (Packagist)
  • Expressive, intuitive syntax.
  • Eloquent ORM, Querybuilder, Migrations
  • Blade template engine + Elixir/Gulp task runner
  • Excellent Documentation and Laracast tutorials

More info at www.laravel.com

A help for Frontend Devs - Elixir

Elixir è un'estensione di Gulp e fornisce una API che semplifica la configurazione del proprio task runner su Laravel.

Installation

npm install

E' già presente un package.json per l'installazione iniziale

Configuration

Si utilizza il gulpfile.js

Configuration

gulpfile.js iniziale


const elixir = require('laravel-elixir');

require('laravel-elixir-vue');

/*
|-----------------------------------------------------------
| Elixir Asset Management
|-----------------------------------------------------------
*/

elixir(mix => {
	mix.sass('app.scss')
	.webpack('app.js');
});

Execution

gulp
gulp --production
gulp watch

Task principali

Compilazione Css

Sass (con LibSass)


// da /resources/assets/sass a /public/css/app.css
elixir(function(mix) {
	mix.sass('app.scss');
});

Si può usare anche RubySass ma installando i pacchetti necessari

Less


// da /resources/assets/less a /public/css/app.css
elixir(function(mix) {
	mix.less('app.less');
});

Compilazione Js

CoffeScript


// da /resources/assets/coffee a /public/js/caffe.js
mix.coffee(['caffe.coffee']);

Browserify


// inclusione di color (npm install color)
elixir.config.js.browserify.transformers.push({
    name: 'color',
    options: {}
});

// da /resources/assets/js/main.js a /public/js/main.js
mix.browserify('main.js');

Compilazione Js

Babel


//da /resources/assets/js/* a /public/js/all.js (come mix.script!)
mix.babel([
        'jsx-component.js',
    ]);

Concatenazione


//Compilazione da /resources/assets/css in /public/css/all.css
mix.styles([
	'module1.css',
	'module2.css'
]);

//Compilazione da /resources/assets/js in /public/js/all.js
//(come mix.babel!)
mix.scripts([
	'module1.js',
	'module2.js'
]);

Si possono concatenare tutti i file di una directory o file da diverse directory specificando le path.

Versioning/Hashing

Aggiunge un hash al filename come cache busting. 

mix.version('css/main.css'); // public/build/css/main-39d5f9a7.css
mix.version('js/app.js'); // public/build/js/app-78efbae6.js

Nei template blade possiamo usare le funzioni di Elixir per recuperare il corretto filename versionato

Task personalizzati?

Certo! con Elixir si ha la possibilità di creare task personalizzati da eseguire. 


elixir.extend('logDate', function() {
	gulp.task('date_logger', function() {
	console.log(new Date());
	});

	// this.registerWatcher('date_logger', '/**/*.js');
	return this.queueTask('date_logger');
});

mix.logDate();
