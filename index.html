In Laravel tutto è già stato organizzato e pronto all'uso.

Come funziona Elixir

Elixir si appoggia su Gulp e aggiunge funzionalità dedicate a Laravel

Predispone da subito una struttura degli assets e ha i task più generici già configurati

Cosa posso fare con Elixir?

  • Compilazione css e js (Sass,Less,PostCss,Coffescript,Babel,Browserify)
  • Concatenazione/minificazione css e js
  • Versioning delle risorse
  • Unit testing php/js
  • Ottimizzazione immagini (via gulp)
  • Linting css/js (via gulp)
  • Livereload (via gulp)

Elixir fornisce:

  • Compilatori sass/less/coffescript/babel/browserify
  • Concatenatori css/js
  • Autoprefixer
  • Versioning
  • PhpUnit Testing

Come sono organizzati gli assets

  • Raw Css in /resources/assets/css
  • Raw Sass in /resources/assets/sass
  • Raw Less in /resources/assets/less
  • Raw Js in /resources/assets/js
  • Public Css in /public/css
  • Public Js in /public/js

Si possono comunque aggiungere/modificare le cartelle (es: /resources/assets/vendor/)

Configurazione di Elixir

Se proprio si vuole modificare la configurazione:

  • assetsDir: la path degli assets, di default è resources/assets
  • cssOutput: la path di output dei css, di default è public/css
  • jsOutput: la path di output dei js, di default è public/js
  • sourcemaps: generare la sourcemap? di default è true

Task principali

Compilazione Css

Sass (con LibSass)


// da /resources/assets/sass a /public/css/app.css
elixir(function(mix) {
	mix.sass('app.scss');
});

Si può usare anche RubySass ma installando i pacchetti necessari

Less


// da /resources/assets/less a /public/css/app.css
elixir(function(mix) {
	mix.less('app.less');
});

Compilazione Js

CoffeScript


// da /resources/assets/coffee a /public/js/caffe.js
mix.coffee(['caffe.coffee']);

Browserify


// inclusione di color (npm install color)
elixir.config.js.browserify.transformers.push({
    name: 'color',
    options: {}
});

// da /resources/assets/js/main.js a /public/js/main.js
mix.browserify('main.js');

Compilazione Js

Babel


//da /resources/assets/js/* a /public/js/all.js (come mix.script!)
mix.babel([
        'jsx-component.js',
    ]);

Concatenazione


//Compilazione da /resources/assets/css in /public/css/all.css
mix.styles([
	'module1.css',
	'module2.css'
]);

//Compilazione da /resources/assets/js in /public/js/all.js
//(come mix.babel!)
mix.scripts([
	'module1.js',
	'module2.js'
]);

Si possono concatenare tutti i file di una directory o file da diverse directory specificando le path.

Versioning/Hashing

Aggiunge un hash al filename come cache busting. 

mix.version('css/main.css'); // public/build/css/main-39d5f9a7.css
mix.version('js/app.js'); // public/build/js/app-78efbae6.js

Nei template blade possiamo usare le funzioni di Elixir per recuperare il corretto filename versionato

Task personalizzati?

Certo! con Elixir si ha la possibilità di creare task personalizzati da eseguire. 


elixir.extend('logDate', function() {
	gulp.task('date_logger', function() {
	console.log(new Date());
	});

	// this.registerWatcher('date_logger', '/**/*.js');
	return this.queueTask('date_logger');
});

mix.logDate();
